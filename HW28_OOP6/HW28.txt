1. Получить рабочее MVC приложение. Если не получится настроить веб-сервер - можно пользоваться сразу обработанными запросами (вводить в браузер строку вида http://localhost/mvc/index.php?url=products/productlist)
Доработать классы ProductList и ProductModel, чтобы страница выдавала реальные данные из базы (будет таблица products с колонками name, price).

2. В папке twig, создать еще один шаблон, вывести в него данные по аналогии с index.html. Сам по себе шаблон будет таблицей с продуктами, данные будут такие же как в ProductList:
$params = [
	'products' => [
		[
			'name' => 'iphone',
			'price' => 400
		],
		[
			'name' => 'samsung',
			'price' => 350
		]
	]
];

3. В MVC, добавить контроллеры по умолчанию. Если, например, пользователь заходит не на
http://localhost/mvc/products/productlist
а на главную страницу (что естественно),
http://localhost/mvc/
его направляет на страницу 404. Нужно сделать, чтобы если пользователь не ввел название экшина или контроллера - это название заменялось строкой 'index'. Т.е. при переходе на
http://localhost/mvc/
должен открыться контроллер Controllers/Index/Index
При переходе на 
http://localhost/mvc/products/
вызывается контроллер Controllers/Products/Index

4. Дописать класс Controllers/Customers/CustomerList.php, чтобы он выводил данные на фронт, в такую же таблицу как с продуктами, с помощью Twig
